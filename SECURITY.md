# Security Policy

## Reporting a Vulnerability

Use a private reporting path for any issue involving:

- tracker credentials
- workflow leakage
- runtime traces or telemetry exposing private operations
- fork attribution or upstream material that appears mis-scoped

Do not file sensitive disclosures as public issues.

## Scope

This public repo should contain only:

- source and tests
- public-safe example workflow material
- curated screenshots
- attribution to upstream specification material
